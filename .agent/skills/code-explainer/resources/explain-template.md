# Code Explanation Template

Use this structure when generating explanations.

---

## [Name of What Is Being Explained]

### What It Does
[One sentence. No jargon. If you must use a technical term, define it in parentheses.]

### How It Works (Step by Step)
1. [First, it does X — like (analogy)]
2. [Then, it does Y — this means (plain-English consequence)]
3. [Finally, it does Z — the result is (what the user sees/gets)]

### Why It Exists
[What problem does this solve? What would happen if we removed it?]

### What It Connects To
- **Uses**: [list of things this code depends on]
- **Used by**: [list of things that depend on this code]

### Analogy
[Compare the entire thing to something from everyday life. Be specific and vivid.]

### Risk Level
[Low / Medium / High — and WHY]

### What BAD Would Look Like
[Show or describe what a poor implementation would look like, and explain the difference. This teaches the orchestrator to spot problems.]
