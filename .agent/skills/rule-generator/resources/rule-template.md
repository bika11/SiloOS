# Rule Template

Use this template when creating a new rule file.

---

```markdown
# [Rule Title — Clear and Descriptive]

## Purpose

[One or two sentences explaining what this rule enforces and why it matters. Write for a non-programmer.]

## Source

[Where this rule came from: user preference, error prevention, project convention detected during bootstrap, etc.]

## Rules

1. [Clear, imperative statement: "Always do X" or "Never do Y"]
2. [Each rule on its own line, numbered]
3. [Be specific — avoid ambiguous language]

## Examples

### Correct (Following This Rule)
[Show an example of behavior that follows this rule]

### Incorrect (Violating This Rule)
[Show an example of behavior that violates this rule and explain why it is wrong]

## Exceptions

[Are there cases where this rule should be relaxed? If none, state "No exceptions."]
```
