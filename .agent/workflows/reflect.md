# /reflect — End-of-Session Review

Use at the end of a work session to consolidate learnings and plan ahead. Type `/reflect`.

## Instructions

### 1. Session Summary
Summarize what was accomplished this session:
- Tasks completed (list them)
- Tasks started but not finished (note what remains)
- Problems encountered and how they were resolved

### 2. Lessons Review
Check `.agent/knowledge/lessons.md`:
- Were any new lessons added this session?
- Were any existing lessons applied? (this validates they are working)
- Are any lessons outdated and should be updated?

### 3. Pattern Review
Check `.agent/knowledge/patterns.md`:
- Were any new patterns logged this session?
- Are any patterns ready for crystallization (3+ occurrences)?
- Propose crystallization for any ready patterns

### 4. Skill Review
Check `.agent/skills/`:
- Were any auto-generated skills used this session? Did they work correctly?
- Do any skills need updating based on what happened?
- Should any new skills be proposed?

### 5. Rule Review
Check `.agent/rules/`:
- Did the orchestrator express any preferences that should become rules?
- Did any rule cause friction or need adjustment?
- Should any new rules be proposed?

### 6. Learning Journal Update
Add entries to `docs/learning-journal.md` for any concepts the orchestrator encountered this session.

### 7. Knowledge File Maintenance
- Check `.agent/knowledge/glossary.md` — any new terms to add?
- Check `.agent/knowledge/project-map.md` — does the map still reflect reality?
- Check `.agent/knowledge/tech-stack.md` — any new technologies added?

### 8. Next Session Recommendations
Suggest 2-3 priorities for the next session:
- Unfinished work from this session
- Patterns ready for crystallization
- Technical debt or improvements noticed
- Any maintenance tasks needed

## Output Format

Keep the reflection concise — the orchestrator should be able to read it in under 2 minutes. Use bullet points, not paragraphs. Only highlight things that are ACTIONABLE or noteworthy.
